Music

This version uses discord.js-lavalink and lavalink by the FredBoat developer.
The quality is expected to be bad due to the bot being hosted on a laptop.
v4 uses ytdl-core, which is limited to only YouTube videos.
However, lavalink has the capability of playing other stuff for music, and even streams!
The only issue is, if the Java server Lavalink crashes, the bot will be unable to play music, and thus return an error.

For the bot to play music, you must be in a voice channel, and then type "||play musiclink". Searches will work like this: "||search youtube searchquery" or "||search soundcloud searchquery".

The bot must have the "CONNECT", "SPEAK" and "USE_VOICE_ACTIVITY" permissions in the voice channel in order to play music.
For looping, type "||loop". Pausing: "||pause". Stopping: "||stop". To set the maximum queue size, type "||setting maxqueuesize 150" (replace 150 with your desired queue size). You require the "MANAGE_SERVER" permission, though.
